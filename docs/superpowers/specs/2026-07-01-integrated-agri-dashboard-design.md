# 智慧农机一体化运营驾驶舱前端改造设计

日期：2026-07-01

## 目标

将现有智能农机前端改造成一个统一的“智慧农机一体化运营驾驶舱”。主体验集中在首页 `/`，融合智慧农业、IoT 监控、地块地图、AI 识别和农机调度 Dashboard 的视觉风格，同时保留现有后端 API、登录鉴权和业务功能。

## 范围

本次只改造前端，不修改后端服务和 API 合约。

保留并重新组织以下现有功能：

- 总览驾驶舱
- 农机/设备管理
- 任务管理
- 控制系统
- AI 视觉识别
- 传感器数据
- 传感器模拟器
- 报表分析
- 登录、登出、角色信息和请求状态处理

## 非目标

- 不新增真实后端接口。
- 不改变认证方式。
- 不删除现有路由页面；主体验集中到 `/`，其他页面可作为兼容入口保留。
- 不引入新的大型 UI 框架。
- 不重写整个项目脚手架。

## 设计方向

采用“单一融合式驾驶舱 + 抽屉/弹窗操作”的方式。

首页作为核心工作台，提供完整运营态势。用户点击设备、任务、控制、AI、传感器、报表等入口时，不跳出主驾驶舱，而是在右侧抽屉、弹窗或面板中完成查看和操作。

## 视觉风格

参考方向：

- ThingsBoard：地图、时序图、告警列表、IoT 设备状态。
- KaaIoT：实时传感器指标卡片。
- EOSDA / OneSoil：地块、卫星、农业风险监控。
- FarmVista / Agrova：浅色绿色农业 SaaS Dashboard。
- Watermetrics：地图地块、灌溉和传感器图表。

视觉规范：

- 主色：农业绿 `#16a34a`。
- 深色：墨绿 `#0f2f23`。
- 背景：浅灰绿和卡片白。
- 状态色：蓝色在线、绿色正常、橙色预警、红色异常。
- 卡片风格：圆角、轻阴影、清晰分区。
- 整体调性：现代农业 SaaS，而不是传统深色大屏。

## 页面布局

主驾驶舱由五个区域组成：

1. 顶部导航栏
   - 品牌名称：云禾智控 / 智慧农机运营驾驶舱。
   - 系统状态、API 状态、实时模式、当前时间。
   - 用户信息、角色、退出登录。
   - 全局刷新、告警入口。

2. 左侧功能导航
   - 总览
   - 设备
   - 任务
   - 控制
   - AI 视觉
   - 传感器
   - 报表
   - 模拟器/演示

3. 中心态势区
   - GIS/地块地图。
   - 农机位置和运行状态。
   - 当前作业地块摘要。
   - AI 识别画面卡片。

4. 右侧运营状态栏
   - 在线设备 TOP 列表。
   - 任务进度。
   - 传感器告警。
   - 关键环境数据。

5. 底部数据分析区
   - 作业面积趋势。
   - 油耗趋势。
   - 产量/收割效率。
   - 传感器时序摘要。
   - 控制状态摘要。

## 交互模型

主页面不频繁跳转。功能入口打开对应抽屉或弹窗：

- 设备抽屉：设备列表、在线状态、详情、定位、启停摘要。
- 任务抽屉：任务列表、进度、地块、设备分配、状态过滤。
- 控制抽屉：姿态、速度、割台、控制量、能力状态。
- AI 弹窗：识别画面、识别类型、置信度、产量分布入口。
- 传感器抽屉：温度、湿度、土壤、水分、告警表格。
- 报表抽屉：作业面积、油耗、效率、事件趋势。
- 模拟器入口：作为演示和调试入口，不作为主视觉重点。

## 数据流

继续使用现有 Pinia store、services 和模型工具。

主驾驶舱通过现有数据聚合逻辑生成 `snapshot`：

- KPI 数据
- 农机数据
- 任务数据
- 传感器数据
- AI 状态
- 告警和事件
- 趋势图数据
- 能力状态文本

页面组件只消费聚合后的展示数据，不直接散落调用 API。这样可以保持现有接口兼容，并便于测试。

## 组件边界

优先复用和改造现有组件：

- `DashboardView.vue`：主驾驶舱容器。
- `DashboardPanel.vue`：统一卡片容器。
- `KpiCard.vue`：指标卡。
- `GisMap.vue`：地图态势区。
- `TrendChart.vue`：趋势图。
- `DashboardTable.vue`：告警、事件、任务列表。

新增或重构的组件建议：

- `IntegratedShell.vue`：统一布局外壳。
- `ActionDrawer.vue`：设备/任务/控制/传感器/报表通用抽屉。
- `OperationSidebar.vue`：左侧功能导航。
- `CommandTopbar.vue`：顶部状态栏。
- `AgriInsightPanel.vue`：底部综合分析卡组。

若时间有限，可以先在 `DashboardView.vue` 内完成第一版，再逐步拆分组件。

## 错误处理和空状态

- 保留现有加载状态和接口错误提示。
- 当后端不可用时，显示明确的错误横幅。
- 空设备、空任务、空告警使用农业场景文案。
- 抽屉内数据为空时不崩溃，显示空状态。
- 认证失效继续回到登录页。

## 响应式要求

优先支持桌面端 1365×768 及以上分辨率。

- 宽屏：三栏布局。
- 中等屏：右侧栏可折叠。
- 小屏：左侧导航收起，抽屉覆盖显示。

## 测试与验证

实施完成后需要验证：

- `npm test` 现有前端测试通过。
- `npm run build` 通过。
- 本地启动后 `/` 页面能显示新驾驶舱。
- 登录页仍可使用 `admin / 20031107` 登录。
- 抽屉入口可以打开和关闭。
- 后端健康状态正常时页面显示在线状态。
- 无后端数据或接口异常时页面不白屏。

## 实施顺序建议

1. 修复现有乱码文案，统一中文显示。
2. 改造首页布局为单一驾驶舱。
3. 统一视觉变量和全局样式。
4. 重组 KPI、地图、AI、告警、趋势区域。
5. 增加设备、任务、控制、传感器、报表抽屉。
6. 保留旧路由兼容入口。
7. 运行测试、构建和浏览器验证。

## 验收标准

- 用户打开 `http://127.0.0.1:3000/index.html#/` 后看到统一的一体化驾驶舱。
- 页面视觉明显融合智慧农业 Dashboard 风格。
- 现有功能入口仍可访问。
- 主要功能通过抽屉/弹窗在单一前端体验内完成。
- 不需要修改后端即可运行。
